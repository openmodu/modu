# modu_code

一个基于 `coding_agent` 的终端 AI 编程助手，采用 Bubble Tea 构建的 TUI 界面。

---

## 快速开始

```bash
go run ./examples/modu_code
```

---

## 键盘快捷键

| 按键 | 说明 |
|------|------|
| `Enter` | 提交消息 |
| `ctrl+c` | 中断当前请求 / 退出 |
| `ctrl+d` | 退出（输入框为空时） |
| `ctrl+l` | 清屏 |
| `ctrl+o` | 切换工具调用展开模式 |
| `esc` | 进入 Vim Normal 模式 |

## Vim 模式

### 为什么要加 Vim 模式？

Vim 模式提供一套纯键盘的浏览方式：一边保留底部输入框和状态栏固定在屏幕底部，一边用熟悉的 `j/k/gg/G` 等按键滚动对话。这样不会因为终端原生滚动把整个 TUI 一起卷走。

要保持底部输入区固定，应该使用应用内部滚动（鼠标滚轮、`j/k`、`pgup/pgdown`、`gg/G`）；不要依赖终端外层的 scrollback/滚动条。

### 使用方式

状态栏左侧始终显示当前模式：`INSERT` 或 `NORMAL`。

**进入 / 退出 Normal 模式**

| 按键 | 说明 |
|------|------|
| `esc` | Insert → Normal |
| `i` 或 `a` | Normal → Insert |

**Normal 模式下的滚动**

| 按键 | 说明 |
|------|------|
| `j` | 向下滚动一行 |
| `k` | 向上滚动一行 |
| `ctrl+d` | 向下半页 |
| `ctrl+u` | 向上半页 |
| `ctrl+f` / `Space` | 向下整页 |
| `ctrl+b` | 向上整页 |
| `G` | 跳到底部 |
| `gg` | 跳到顶部 |

**Normal 模式下的复制（Yank）**

| 按键 | 说明 |
|------|------|
| `yy` | 复制最后一条 AI 回复到剪切板 |
| `yG` | 复制整个对话到剪切板 |

> 复制成功后状态栏会显示提示。当前仅支持 macOS（使用 `pbcopy`）。
