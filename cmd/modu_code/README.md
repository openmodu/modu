# modu_code

一个基于 `coding_agent` 的终端 AI 编程助手，采用 Bubble Tea 构建的 TUI 界面。

---

## 快速开始

```bash
go run ./examples/modu_code
```

---

## 键盘快捷键

| 按键 | 说明 |
|------|------|
| `Enter` | 提交消息 |
| `ctrl+c` | 中断当前请求 / 退出 |
| `ctrl+d` | 退出（输入框为空时） |
| `ctrl+l` | 清屏 |
| `ctrl+o` | 切换工具调用展开模式 |
| `esc` | 进入 Vim Normal 模式 |

## Vim 模式

### 为什么要加 Vim 模式？

Vim 模式分成两种终端行为：
- `INSERT` 下由应用接管鼠标滚轮，适合在固定 footer 的前提下浏览对话。
- `NORMAL` 下把鼠标交还给终端，适合直接拖拽选择和复制文本。

如果要保持底部输入区固定，应该使用应用内部滚动；如果进入 `NORMAL` 后再使用终端外层 scrollback/滚动条，整个 TUI 都会一起移动。

### 使用方式

状态栏左侧始终显示当前模式：`INSERT` 或 `NORMAL`。

**进入 / 退出 Normal 模式**

| 按键 | 说明 |
|------|------|
| `esc` | Insert → Normal |
| `i` 或 `a` | Normal → Insert |

**Normal 模式下的滚动**

| 按键 | 说明 |
|------|------|
| `j` | 向下滚动一行 |
| `k` | 向上滚动一行 |
| `ctrl+d` | 向下半页 |
| `ctrl+u` | 向上半页 |
| `ctrl+f` / `Space` | 向下整页 |
| `ctrl+b` | 向上整页 |
| `G` | 跳到底部 |
| `gg` | 跳到顶部 |

**Normal 模式下的复制（Yank）**

| 按键 | 说明 |
|------|------|
| `yy` | 复制最后一条 AI 回复到剪切板 |
| `yG` | 复制整个对话到剪切板 |

进入 `NORMAL` 后，也可以直接用鼠标拖拽选择终端里的文字进行复制。

> `yy` / `yG` 复制成功后状态栏会显示提示。当前仅支持 macOS（使用 `pbcopy`）。
