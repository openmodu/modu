# modu_code

一个基于 `coding_agent` 的终端 AI 编程助手，采用 Bubble Tea 构建的 TUI 界面。

---

## 快速开始

```bash
go run ./examples/modu_code
```

---

## 键盘快捷键

| 按键 | 说明 |
|------|------|
| `Enter` | 提交消息 |
| `ctrl+c` | 中断当前请求 / 退出 |
| `ctrl+d` | 退出（输入框为空时） |
| `ctrl+l` | 清屏 |
| `ctrl+o` | 切换工具调用展开模式 |
| `esc` | 进入 Vim Normal 模式 |

## Vim 模式

### 为什么要加 Vim 模式？

TUI 界面使用了 `tea.WithMouseCellMotion()` 来支持鼠标滚轮滚动 viewport。但这带来一个问题：**鼠标事件被 TUI 程序捕获后，终端原生的文本选中和复制功能就失效了**——用户无法用鼠标拖选 AI 的回复内容进行复制。

常见的解法是：
- 按住 `⌥ Option`（macOS）拖选绕过捕获——需要用户记忆额外操作
- 加一个切换按键在"滚动模式"和"选择模式"之间手动切换——割裂的体验

Vim 模式提供了更自然的解决方式：进入 Normal 模式时自动关闭鼠标捕获，此时终端恢复原生文本选中；回到 Insert 模式时自动恢复鼠标滚轮。对于 Vim 用户来说，这套模式切换是肌肉记忆，完全无需额外学习。

### 使用方式

状态栏左侧始终显示当前模式：`INSERT` 或 `NORMAL`。

**进入 / 退出 Normal 模式**

| 按键 | 说明 |
|------|------|
| `esc` | Insert → Normal |
| `i` 或 `a` | Normal → Insert |

**Normal 模式下的滚动**

| 按键 | 说明 |
|------|------|
| `j` | 向下滚动一行 |
| `k` | 向上滚动一行 |
| `ctrl+d` | 向下半页 |
| `ctrl+u` | 向上半页 |
| `ctrl+f` / `Space` | 向下整页 |
| `ctrl+b` | 向上整页 |
| `G` | 跳到底部 |
| `gg` | 跳到顶部 |

**Normal 模式下的复制（Yank）**

| 按键 | 说明 |
|------|------|
| `yy` | 复制最后一条 AI 回复到剪切板 |
| `yG` | 复制整个对话到剪切板 |

> 复制成功后状态栏会显示提示。当前仅支持 macOS（使用 `pbcopy`）。

**Normal 模式下选中文本**

进入 Normal 模式后，鼠标捕获自动关闭，可以直接用鼠标拖选任意内容，然后 `Cmd+C` 复制。

## 状态栏说明

状态栏始终显示在底部最后一行：

```
INSERT                                    42%
```

| 区域 | 内容 |
|------|------|
| 左侧 | 当前模式（`INSERT` / `NORMAL`）+ 状态消息 |
| 右侧 | 滚动位置百分比（未在底部时显示） |
